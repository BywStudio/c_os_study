# 手写操作系统学习计划

## 前置知识要求

### 1. C 语言进阶（2-3 周）

你已有 C 语言基础，但写 OS 需要更深入的理解：

- **指针与内存管理**：多级指针、指针运算、函数指针
- **位操作**：位移、与或非、位域（硬件寄存器操作必备）
- **volatile 关键字**：防止编译器优化掉硬件访问
- **内联汇编**：`asm` 语法，C 与汇编混合编程
- **链接与加载**：理解 ELF 格式、链接脚本（linker script）
- **结构体对齐与打包**：`__attribute__((packed))`，硬件数据结构要求精确内存布局

**推荐资源**：
- 《C 程序设计语言》（K&R）
- 《C 专家编程》

### 2. x86 汇编语言（2 周）

操作系统的启动代码必须用汇编：

- **NASM 语法**（推荐）或 AT&T 语法
- **实模式 / 保护模式 / 长模式**的寄存器与指令集差异
- **中断与异常处理机制**
- **栈帧管理**

**推荐资源**：
- 《x86 汇编语言：从实模式到保护模式》李忠（强烈推荐，与 OS 开发高度匹配）
- NASM 官方文档

### 3. 计算机体系结构基础（1-2 周）

不需要深入硬件设计，但必须理解 CPU 如何工作：

- **CPU 执行模型**：取指-译码-执行
- **内存层级**：寄存器 → Cache → RAM → 磁盘
- **MMU 与分页机制**：虚拟地址 → 物理地址转换
- **中断控制器（PIC/APIC）**：硬件中断的工作方式
- **DMA**：设备直接内存访问
- **I/O 端口与 MMIO**：CPU 与设备通信的两种方式

**推荐资源**：
- 《计算机系统要素》（Nand2Tetris）
- Intel 64 and IA-32 Architectures Software Developer's Manual（参考用，不需要通读）

### 4. 操作系统原理（2-3 周）

理解 OS 的核心概念，才能知道要实现什么：

- **引导过程**：BIOS → MBR → Bootloader → Kernel
- **进程/线程模型**：上下文切换、调度算法
- **内存管理**：分页、堆分配、虚拟内存
- **文件系统**：FAT12/16、简单文件系统设计
- **系统调用**：用户态 ↔ 内核态切换（int 0x80 / syscall）
- **设备驱动模型**

**推荐资源**：
- 《操作系统导论》（OSTEP）— 最易读的 OS 教材
- 《操作系统真象还原》郑钢 — 中文，实战导向

## 分阶段实战计划

### 阶段一：搭建环境 + Hello World（1 周）

**目标**：能启动一个最简单的内核并在屏幕上打印字符

- 安装工具链：`nasm`、`qemu`、`gcc` 交叉编译器（`i686-elf-gcc` 或 `x86_64-elf-gcc`）
- 编写 MBR 引导扇区（512 字节，0x55AA 结尾）
- 通过 BIOS 中断 `int 10h` 在实模式下打印 "Hello OS"
- 用 QEMU 启动验证

**产出**：一个从 QEMU 启动并打印字符串的镜像

### 阶段二：进入保护模式（1-2 周）

**目标**：从 16 位实模式切换到 32 位保护模式

- 理解 GDT（全局描述符表）的构造与加载
- A20 地址线开启
- CR0 寄存器设置，切换到保护模式
- 在保护模式下通过显存地址 `0xB8000` 直接写屏

**产出**：在保护模式下运行代码并显示输出

### 阶段三：内核基础设施（2-3 周）

**目标**：建立内核运行的基本框架

- 编写链接脚本，规划内核内存布局
- 实现 VGA 文本模式驱动（printf 功能）
- 实现 GDT / IDT 设置
- 中断处理框架（键盘中断作为第一个实战）
- 基础端口 I/O 函数（`inb` / `outb`）

**产出**：能响应键盘中断并打印按键的内核

### 阶段四：内存管理（2-3 周）

**目标**：实现物理内存和虚拟内存管理

- 物理内存检测（通过 BIOS int 15h 或 multiboot 信息）
- 物理页帧分配器（位图法或链表法）
- 页目录 / 页表设置，启用分页
- 内核堆分配器（简单实现 kmalloc / kfree）

**产出**：能分配和释放内存的内核

### 阶段五：进程管理（3-4 周）

**目标**：实现多任务切换

- 定义进程控制块（PCB）
- 内核线程的创建与切换
- 上下文切换（保存/恢复寄存器状态）
- 简单轮转调度器
- 用户态进程：特权级切换（Ring 0 → Ring 3）
- 系统调用框架（通过 int 0x80 或 sysenter）

**产出**：能在多个任务间切换的内核

### 阶段六：文件系统（2-3 周）

**目标**：实现一个简单的文件系统

- 磁盘驱动（IDE / ATA PIO 模式读写扇区）
- 实现简单文件系统（可参考 FAT12 或自己设计）
- 文件系统 API：open / read / write / close
- 将内核从硬盘中加载用户程序执行

**产出**：能从磁盘读取文件并执行用户程序的内核

### 阶段七：用户态与 Shell（2 周）

**目标**：实现一个可交互的用户界面

- 用户程序加载与执行
- 命令行 Shell
- 基础命令：ls、cat、echo 等

**产出**：一个有 Shell 界面、可执行程序的小型操作系统

## 预估总时间

按每天 2-3 小时学习计算：

| 阶段 | 时间 |
|------|------|
| 前置知识补充 | 6-10 周 |
| 阶段一~二 | 2-3 周 |
| 阶段三~四 | 4-6 周 |
| 阶段五~七 | 7-9 周 |
| **总计** | **约 19-28 周（5-7 个月）** |

> 不必等前置知识全部学完再动手。建议在学习汇编和体系结构时同步开始阶段一，边做边学效果最好。
