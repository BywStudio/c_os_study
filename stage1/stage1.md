## 2. 创建 MBR 引导扇区

> 什么是 MBR 引导扇区？
>
> - Master Boot Record，主引导记录，是硬盘启动的第一个扇区，共 512 字节
>
> - MBR 是 BIOS 交出控制权后，CPU 执行的第一段用户代码，是操作系统启动的起点

> 计算机启动流程：
>
> 1. BIOS 上电自检（POST）
> 2. BOIS 在硬盘上寻找第一个扇区（MBR）
> 3. 检查最后两字节是否为 0x55AA，是，则认为可以启动
> 4. 把这 512 字节加载到内存的 0x7C00 处，跳转过去执行

MBR 的 512 字节结构

| 区域     | 大小     | 内容                                |
| -------- | -------- | ----------------------------------- |
| 引导代码 | 446 字节 | 可执行的机器指令（我们的 boot.asm） |
| 分区表   | 64 字节  | 记录硬盘分区信息                    |
| 启动标志 | 2 字节   | 固定 0x55AA，BIOS 靠它识别可启动    |

> 什么是通用寄存器？
>
> CPU 内部的一组高速存储单元，用于暂存指令执行时需要的数据
>
> - 可以存放任意类型的数据（地址、数值、中间结果等）
>
> - 没有固定用途限制，由程序员自由决定存放内容
> - x86 有 4 个 16 位通用寄存器：AX、BX、CX、DX
> - 可以理解为 CPU 自带的口袋，存取速度比内存 RAM 快几百倍，但容量极小（每个只有几字节）
> - CPU 执行指令时，必须先把数据从内存读到寄存器，再对寄存器中的数据进行运算
>   - 如：`mov ax 0x07C0` 就是把数字 0x07C0 放到 AX 这个口袋中，后续指令直接从 AX 取值
>
> 通用寄存器：CPU 中可以存放任意类型的寄存器，既能存地址，也能存数值，没有固定用途的限制
>
> 专用寄存器：有固定用途，如 sp（栈指针）、ip（指令指针）、ds 数据段等

> 什么是专用寄存器？
>
> 专用寄存器是 CPU 中具有固定用途的寄存器，只能用于特定的功能，不能由程序员自由决定存放内容

专用寄存器：

| 寄存器 | 全称                | 固定用途                           |
| ------ | ------------------- | ---------------------------------- |
| SP     | Stack Pointer       | 栈指针，指向栈顶                   |
| BP     | Base Pointer        | 基址指针，指向栈帧基址             |
| IP     | Instruction Pointer | 指令指针，指向下一条要执行的指令   |
| SI     | Source Index        | 源变址，字符串操作时指向源数据     |
| DI     | Destination Index   | 目标变址，字符串操作时指向目标位置 |
| CS     | Code Segment        | 代码段，取指令时用                 |
| DS     | Data Segment        | 数据段，读写数据时用               |
| SS     | Stack Segment       | 栈段，栈操作时用                   |
| ES     | Extra Segment       | 附加段，额外数据访问               |
| FLAGS  | Flags Register      | 标志寄存器，存放 CPU 状态标志      |

通用寄存器：

| 16 位 | 名字       | 默认用途             | 实际用途 |
| ----- | ---------- | -------------------- | -------- |
| AX    | 累加器     | 算术运算、I/O 操作   | 任意     |
| BX    | 基址寄存器 | 内存寻址基址         | 任意     |
| CX    | 计数寄存器 | 循环/重复计数        | 任意     |
| DX    | 数据寄存器 | I/O 端口地址、乘除法 | 任意     |

> 什么是段寄存器？
>
> 是 x86 架构中的一个16 位段寄存器，用于存放数据段的段基地址
>
> - 在实模式下，当指令需要访问内存中的数据时，CPU 默认以 DS 的值作为段地址
> - 结合指令中给出的偏移地址，计算出实际的物理地址（物理地址 = DS x 16 + 偏移）
>
> 代码中，设置的`DS = 0x07C0`，lodsb 指令从 DS:SI 读取数据，实际物理地址 = 0x07C0 x 16 + SI
>
> 如果不设置 DS，CPU 会使用默认值，读取的地址就是错的，打印出来是乱码
>
> 简单类比：
> - DS = 书架编号，告诉你去哪个书架找
> - 偏移（如 SI）= 第几层第几本，告诉你在书架上的具体位置

段寄存器：

| 段寄存器 | 全称          | 负责的内存访问类型    |
| -------- | ------------- | --------------------- |
| CS       | Code Segment  | 取指令                |
| DS       | Data Segment  | 读写数据              |
| SS       | Stack Segment | 栈操作（push/pop）    |
| ES       | Extra Segment | 附加数据段            |
| FS       | -             | 附加段（32 位后新增） |
| GS       | -             | 附加段（32 位后新增） |

使用到的指令

| 指令  | 全称             | 作用                                 |
| ----- | ---------------- | ------------------------------------ |
| org   | Origin           | 设置程序起始偏移地址                 |
| mov   | Move             | 将数据从源位置复制到目标位置         |
| lodsb | Load String Byte | 从 DS:SI 读 1 字节到 AL，SI 自动 +1  |
| cmp   | Compare          | 比较两个值，设置标志位（不保存结果） |
| je    | Jump if Equal    | 相等则跳转                           |
| int   | Interrupt        | 触发指定中断号的中断服务             |
| jmp   | Jump             | 无条件跳转                           |
| hlt   | Halt             | CPU 停止执行，进入低功耗状态         |
| db    | Define Byte      | 定义字节序列数据                     |
| dw    | Define Word      | 定义 2 字节数据                      |
| times | -                | 重复执行指定次数                     |

| 参数        | 类型            | 说明                                                       |
| ----------- | --------------- | ---------------------------------------------------------- |
| 0x7C00      | 立即数          | BIOS 加载 MBR 的内存地址                                   |
| 0x07C0      | 立即数          | 段地址值（0x7C00 / 16）                                    |
| ax          | 16 位通用寄存器 | 累加器，常用作中转                                         |
| ds          | 段寄存器        | 数据段，数据访问基于 DS:偏移                               |
| es          | 段寄存器        | 附加段，BIOS 中断可能用到                                  |
| ss          | 段寄存器        | 栈段，配合 SP 确定栈位置                                   |
| sp          | 16 位通用寄存器 | 栈指针，指向栈顶                                           |
| si          | 16 位通用寄存器 | 源变址寄存器，指向要读取的数据地址                         |
| al          | 8 位通用寄存器  | AX 的低 8 位，存放读入的字符                               |
| ah          | 8 位通用寄存器  | AX 的高 8 位，存放 BIOS 中断功能号                         |
| bh          | 8 位通用寄存器  | BX 的高 8 位，表示显示页号                                 |
| bl          | 8 位通用寄存器  | BX 的低 8 位，表示字符颜色属性                             |
| 0x0E        | 立即数          | int 10h 功能号，电传打字模式                               |
| 0           | 立即数          | 字符串结束符 / 显示页号第 0 页                             |
| 0x07        | 立即数          | 字符颜色属性（高 4 位 0 = 黑色背景，低 4 位 7 = 白色前景） |
| 0x10        | 立即数          | BIOS 显示服务中断号                                        |
| 0xAA55      | 立即数          | MBR 启动标志（小端序存储为 55 AA）                         |
| msg         | 标签            | 字符串首地址偏移量                                         |
| .print_loop | 标签            | 打印循环入口                                               |
| .done       | 标签            | 打印结束入口                                               |
| $           | NASM 符号       | 当前偏移地址                                               |
| $$          | NASM 符号       | 段起始偏移地址                                             |

```nasm
; 设置程序起始偏移地址为 0x7C00（BIOS 加载 MBR 的地址）
[org 0x7C00]

; 将段地址值 0x0000 装入 ax（配合 org 0x7C00，段地址从 0 开始）
mov ax, 0x0000

; 设置数据段 DS = 0x0000（不能直接写立即数到段寄存器）
mov ds, ax

; 设置附加段 ES = 0x0000（BIOS 中断内部可能用到）
mov es, ax

; 设置栈段 SS = 0x0000
mov ss, ax

; 设置栈指针 SP = 0x7C00（栈向下增长，代码向上增长，互不干扰）
mov sp, 0x7C00

; 将字符串首地址偏移量装入 SI，准备逐字符读取
mov si, msg

.print_loop:
    ; 从 DS:SI 读 1 字节到 AL，SI 自动 +1
    lodsb

    ; 比较 AL 和 0（0 = 字符串结束符）
    cmp al, 0

    ; 如果 AL == 0，跳转到 .done（字符串打印完毕）
    je .done

    ; 设置 BIOS 中断功能号 0x0E（电传打字模式，输出字符并前进光标）
    mov ah, 0x0E

    ; 设置显示页号 = 0（第 0 页）
    mov bh, 0

    ; 设置字符颜色属性（高 4 位 0 = 黑色背景，低 4 位 7 = 白色前景）
    mov bl, 0x07

    ; 调用 BIOS 显示服务中断 0x10，输出 AL 中的字符到屏幕
    int 0x10

    ; 跳回循环开头，继续打印下一个字符
    jmp .print_loop

.done:
    ; 让 CPU 停止执行，进入低功耗状态
    hlt

    ; 如果被意外唤醒，跳回 hlt 继续停止
    jmp .done

; 定义以 0 结尾的字符串数据，标签 msg 指向首地址
msg: db "Hello OS", 0

; 用 0 填充到第 510 字节（$ - $$ = 已生成字节数）
times 510 - ($ - $$) db 0

; 写入 MBR 启动标志（小端序存储为 55 AA，BIOS 靠它识别可启动扇区）
dw 0xAA55

; ============================================================
; 用法
; ============================================================
;
; 编译：
;   nasm -f bin boot.asm -o boot.bin
;
; 制作软盘镜像：
;   dd if=/dev/zero of=floppy.img bs=512 count=2880
;   dd if=boot.bin of=floppy.img bs=512 count=1 conv=notrunc
;
; 在 QEMU 中运行：
;   qemu-system-i386 -fda floppy.img
;
; 或使用 Makefile：
;   make        编译并制作镜像
;   make run    编译并在 QEMU 中启动
;   make clean  清理生成文件
```

`Makefile`文件内容

- 文件作用：构建脚本

> 什么是 make？
>
> make 是一个自动化构建工具，读取 Makefile 文件中定义的规则
>
> - 自动判断哪些文件需要重新编译，然后执行对应的命令
> - 核心逻辑是：比较目标文件和依赖文件的时间戳，如果依赖文件比目标文件新，就重新执行构建命令
>
> 举例：
>
> - 执行 make run 时，发现 boot.asm 比 boot.bin 新，就自动运行 nasm 编译
> - 如果 boot.asm 没改过，就跳过编译直接启动 QEMU

> 什么是 Makefile？
>
> Makefile 是 make 工具的构建配置文件，用于定义项目中文件之间的依赖关系和构建规则
>
> - 通过 make 命令自动执行编译、链接、打包等操作，避免每次手动输入长串命令

make 常用的命令和参数：

| 命令             | 作用                                               |
| ---------------- | -------------------------------------------------- |
| `make`           | 执行 Makefile 中的默认目标（通常是第一个目标）     |
| `make 目标名`    | 执行指定目标，如 `make run`、`make clean`          |
| `make -n`        | 只显示要执行的命令但不真正执行（模拟运行）         |
| `make -B`        | 强制重新构建所有目标，不管文件是否更新             |
| `make -j N`      | 并行构建，N 为并发数（如 `make -j4`）              |
| `make -f 文件名` | 指定 Makefile 文件名，默认读取名为 Makefile 的文件 |
| `make -C 目录`   | 切换到指定目录再执行 make                          |

| 参数        | 作用                                         |
| ----------- | -------------------------------------------- |
| `-n`        | 干跑模式，只打印命令不执行                   |
| `-B`        | 无条件重新构建所有目标                       |
| `-f 文件名` | 指定构建文件名                               |
| `-C 目录`   | 指定工作目录                                 |
| `-j N`      | 并行执行 N 个任务                            |
| `-k`        | 遇到错误继续构建其他目标                     |
| `-s`        | 静默模式，不打印执行的命令                   |
| `VAR=值`    | 命令行覆盖变量，如 `make NASM=/usr/bin/nasm` |

Makefile 中常用指令：

| 指令             | 作用                                           |
| ---------------- | ---------------------------------------------- |
| `.PHONY`         | 声明伪目标，告诉 make 该目标不对应实际文件     |
| `all`            | 默认目标，通常作为第一个目标，依赖其他所有目标 |
| `clean`          | 清理目标，删除所有生成文件                     |
| `install`        | 安装目标，将编译产物安装到系统目录             |
| `test`           | 测试目标，运行测试用例                         |
| `run`            | 运行目标，编译并启动程序                       |
| `help`           | 帮助目标，显示可用目标及说明                   |
| `变量 = 值`      | 变量定义，如 `CC = gcc`                        |
| `$(变量)`        | 变量引用，取变量的值                           |
| `目标: 依赖`     | 规则定义，声明目标和依赖关系                   |
| `\t命令`         | 构建命令，Tab 缩进，依赖满足后执行             |
| `include 文件名` | 包含其他 Makefile 文件                         |
| `ifeq / endif`   | 条件判断，根据条件选择不同构建逻辑             |
| `%.o: %.c`       | 模式规则，通配匹配同类文件的构建方式           |
| `@命令`          | 静默执行，不打印该命令本身                     |
| `-命令`          | 忽略错误，命令失败不中断构建                   |

```makefile
# 引导扇区二进制输出文件名
BOOT_BIN = boot.bin

# 软盘镜像输出文件名
FLOPPY_IMG = floppy.img

# 汇编源代码文件名
ASM_SRC = boot.asm

# 汇编器可执行文件名
NASM = nasm

# QEMU 模拟器（i386 = 模拟 32 位 x86）
QEMU = qemu-system-i386

# 磁盘操作工具
DD = dd

# 默认目标：编译并制作镜像
all: $(FLOPPY_IMG)

# 汇编 boot.asm -> boot.bin
$(BOOT_BIN): $(ASM_SRC)
        # -f bin = 纯二进制格式，-o = 输出文件名
        $(NASM) -f bin $(ASM_SRC) -o $(BOOT_BIN)

# 制作软盘镜像
$(FLOPPY_IMG): $(BOOT_BIN)
        # 创建 1.44MB 空白软盘镜像（2880 × 512 = 1.44MB）
        $(DD) if=/dev/zero of=$(FLOPPY_IMG) bs=512 count=2880
        # 将引导扇区写入第 0 扇区，notrunc = 不截断目标文件
        $(DD) if=$(BOOT_BIN) of=$(FLOPPY_IMG) bs=512 count=1 conv=notrunc

# 在 QEMU 中启动
run: $(FLOPPY_IMG)
        # -fda = 从软盘镜像启动
        $(QEMU) -fda $(FLOPPY_IMG) -vga std

# 清理生成文件
clean:
        # -f = 强制删除，文件不存在也不报错
        rm -f $(BOOT_BIN) $(FLOPPY_IMG)

# 声明伪目标（不对应实际文件，避免与同名文件冲突）
.PHONY: all run clean

# ============================================================
# 用法
# ============================================================
#
# make        编译生成 boot.bin 和 floppy.img
# make run    编译并在 QEMU 中启动运行
# make clean  清理所有生成文件
```

执行`make run`，这条指令会做：

- 用 NASM 编译 boot.asm → boot.bin

- 用 dd 创建 1.44MB 空白软盘镜像 floppy.img

- 将 boot.bin 写入软盘镜像第一个扇区

- 启动 QEMU 运行镜像

如果之前的代码有错误，可以修改代码后 `make clean && make run`
